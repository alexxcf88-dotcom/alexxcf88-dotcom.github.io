No hace falta inventar porcentajes.

Antes de pedir una demo, la clínica debería poder hacerse la cuenta y resolver las dudas normales: llamadas perdidas, datos, agenda, fallos y permanencia.

Llamadas ¿Cuántas llamadas no cogéis cada semana?

Fuera de horario, a la hora de comer o mientras recepción atiende a un paciente.

WhatsApp ¿Cuántos WhatsApps quedan para responder luego?

Una duda sencilla puede enfriarse si la respuesta llega horas después.

Revisiones ¿Cuántos pacientes llevan más de 6 meses sin volver?

Revisiones, limpiezas y tratamientos pendientes que ya están en vuestra base.

¿Y si el paciente quiere hablar con una persona?

Se deriva a recepción. AItomat no fuerza el cierre: pregunta, prepara el contexto y pasa el caso cuando el paciente lo pide o cuando el motivo lo requiere.

¿Se presenta como IA?

Sí. En llamadas se identifica como asistente de la clínica. No se vende como sustituto humano indistinguible, sino como una recepción siempre disponible.

¿Qué pasa con los datos de pacientes?

Los datos de salud requieren especial cuidado. Antes de conectar canales reales se firma contrato de encargado de tratamiento y se define qué datos se tratan, dónde y con qué medidas.

¿Tengo que cambiar mi software de agenda?

No como punto de partida. Se revisa vuestra agenda actual y se conecta lo viable. Para pilotos, Google Calendar está contemplado; otras agendas se valoran caso a caso.

¿Y si falla o no entiende al paciente?

El flujo debe tener salida humana. Si hay duda, urgencia, datos contradictorios o petición sensible, el caso se marca para recepción en lugar de improvisar.

¿Hay permanencia?

No. La propuesta base es mensual y sin permanencia. Si una integración requiere trabajo específico, se explica antes de activarla.
